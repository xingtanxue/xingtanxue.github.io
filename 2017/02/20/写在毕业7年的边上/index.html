写在毕业7年的边上

发表于 2017-02-20 | 更新于 2017-02-20 | |

博主于2010年正式毕业于一所理工科院校的软件专业，全国首批200所软件学院之一，位于美丽的中原城市、“四圣”故里-南阳。读这所学校的原因在于一个不服输的文科生就读理科的结果，高考离理想的一本有50分之差，在高考补录期间，考虑到家庭情况和父母的建议，便留在家乡，学习Java、Html、数据库、软件设计、数据结构、Ajax、SSH等软件开发课程。期间参加足球社团，总共踢了3次球；学院田径队，每天早上5000米训练，现在的好身体利益于当初的早起长跑；冲过四级，一直在学院的实习基地参与Java项目开发。

在毕业1年后在深圳正式进入Android开发领域，当时还买了一部华为U8500-第一步智能手机，干了2年多，2013年转战杭州，至今已将近4年；在深圳工作时一身西装，到杭州转成便装，如今再穿上皮鞋，刚从上家公司离职，准备寻找下一份工作机会。坦言我是个不擅选择的人，每次公司发offer即停止面试开启新的工作，大概面试时将该谈的都谈过，大家都能接受就可以，4次工作3次都是这样，其中第3次工作是一次外派项目，更换项目、更换地点，每次都弄的焦头烂额，所以提醒大家选择机会时，如果遇到这样的工作一定要慎重，除非你不在乎职业发展想全国到处去看看；当然第4次工作时，就转换过来，负责Android组项目架构，一直干了3年，自己不断提升项目开发效率和质量，公司业务却起色不大，但至今不能说离职是不是一个对的选择，创业公司本身就在不断的转换方向，黑暗里寻找商机，万一碰对了起飞即在眼前；但离开即意味着新的开始，在此做一个总结，对未来做一个瞻望，还是很有必要的。

博主有“处女座情怀”，对代码结构、注释类型、算法结构有要求，对性能、内存也会做些要求，对项目结构、基础控制有些要求，甚至对开发期间提交代码的次数也要求每天1次，在开发人员少的情况下可以约束，慢慢的就需要写出规范文档进行培训，对代码提交进行Review；逐渐人员增多，功能也增多后，要求自测的同时，只得对核心功能进行Review，其他交给质量管控人员，设置模块负责人和代码互相Review也是很有必要的。做开发的第一步是Show me the Code，专业能力不断的提升，知识面不断的拓展，前3年做的还不错，领导担心的问题都没有出现，由于积极主动、工作态度良好，一度受到领导好评；但在沟通上出现一些情绪问题，当时存在不认同非工作伙伴的问题，沟通不在一个频道上，自然容易出问题，后来在第4份工作得到比较大改观。

但逐渐自身年龄增长和阅历的增长，年轻人慢慢加入团队成为主要力量，由于招聘机制的不完善，这些人做事的态度和专业能力的欠缺可能造成新的沟通问题，问题形成矛盾，如果不能很好的解决会造成互相不认同，工作无法有效开展的情况；在离职之前尽量配合和迁就彼此的工作，自己能做的多做些避免短时间内出现沟通问题；在离职之后发现，其实跟最初加入团队时，成员结构已经发生天翻地覆的变化，原成员几乎换一遍，自己成为唯一“幸存者”，3年以上的招人标准降到毕业生亦可，中坚力量发生变化，而自己未增改变，造成问题的出现，公司的业务不见起色以及身边同事的吐槽，同样成为萌生离职念头的原因；其实重要的是第1个原因和第3个原因，以前周边环境未发生变化则可从容面对，当发生变化而自身未变化，则问题产生，这是内部原因；身边同事的不断吐槽使得对公司业务的“悲观”情绪变深，则是外部原因，在职场深切感知这些“负力场”的负面作用，以后尽量远离他们才好，听同事吐槽不是件好事情，“相信则成，不信则散”，大家集共同的信仰到同一家公司，应该同进退，只有如此才能享受风雨同舟的战斗友情，无共同信仰最好离开，自己不爽还让别人也不爽，这些人该怎样评价呢？有人的地方就有江湖，有江湖的地方就有过的好和不好，有怨言是可能的，但要及时自己消化和排遣是很重要的，讲出来不仅加深自己的怨念，同时影响别人，对任何人都没有好处，这种事最好对花讲、对树讲、对河讲，让情绪随自然界的风而去，把美好的事情和心情装进来，重新出发。

另一方面到达30岁的年纪，回家看到父母的老迈，大家对自己新的期待，感受到时间力量的沉重，自己不再是青葱少年，可以任意妄为的年代，说走就走的旅行只好等到50岁时再享受，只希望到时父母依然健在，身体健康。上面咱们已经说过排遣的重要性，一个不懂得的人是不会有大成就的，个人能力往往需要透过群体进行放大，才能取得更大成就；选择跟谁做朋友也同样重要，周边的人对自己的影响比较大，成为生活环境、生活条件的必备因素，亲近积极、远离消极的人，少说多做、不要胡言乱语、专注自身岗位的职责，在专业能力完备的同时，这些软条件也同样变得越来越重要；工作环境怎样，都是人造成的，好的品德可以影响周围的人，厚德载物，求真至善，尔后注重自身品德的修养显得尤其重要；避免骄傲和愤怒的情绪是首先需要注意的，骄傲让人自视甚高，容易脱离大家，成功时恶人，失败时恶己；愤怒不能解决问题，反而会给人留下坏印象，不成熟也不专业，显得难以沟通，往往会使自己孤立。

写完这个，再写写自己的求职方向，大概是成长型的公司或者是稳定型的公司；专业方面，基础知识，底层原理，模式理论熟悉的七七八八，主要是RN、C语言、Python再深入学习下，Jni、底层修改编译、项目周边再熟悉下，Ios有机会学习下，写本Android基础教材，2017年就定这么多吧。
